### 开发

只需要执行以下命令就可以在 <http://localhost:3000> 中看到

```bash
建议使用 pnpm
node > 18
pnpm dev
```

### 构建

构建该应用只需要执行以下命令

```bash
pnpm build
```

然后你会看到用于发布的 `dist` 文件夹被生成。



## 开发规范及项目说明


### 1、项目结构说明

组建必须放在components文件夹下面，否则不负你自动导入
  #### 项目结构
  - 本项目为了抽离公共的封装，有4个子仓库，位于src目录：01-kk-system、02-kk-i18n-admin、02-kk-i18n-system、04-kk-component-mobile（移动端公共组件）或者04-kk-component（pc端公共组件）
  <br />

  - 01-kk-system：项目基础工具
    + allHttp：http请求的封装以及各个系统的api接口目录
    + allDefine：共公的常量定义
    + allUtils：公共的工具类定义
    + middleware：提供多个接口融合的业务数据接口，以及提供需要缓存的网络接口
    **修改注意：【工具类文件】、【常量文件】、【http的api文件】可以新增，如果需要修改，请考虑对现有项目的影响，如不确定可以联系管理员**
  <br />

  - 02-kk-i18n-admin：各个子系统的多语言文件，每个系统独有的多语言文件，单独放在这个文件夹下面。命名方式：系统名目录/多语言目录/多语言定义的js文件。可以阅读目录下的readme文件了解。【需要自己通过i18n引入自己项目需要的多语言文件，不需要就不用引入】
  **修改注意：
    1、新增： 新增的自由项目多语言文件独立一个单独的文件夹，命名结合业务明确定义。
    2、修改：如无特殊需求，请不要修改已有多语言文件内容。如果需要修改，请考虑对现有项目的影响，如不确定可以联系管理员**
  <br />

  - 02-kk-i18n-system：全部系统公共的多语言文件，以及UI库的多语言（我们多语言种类笔ui库多，需要提出来自己做翻译）
    + system：全部系统公共的多语言文件，新建项目需要通过i18n引入的文件;
    + uiLib：pc端UI库element plus的多语言文件，pc项目必须要通过i18n引入；
    + uiLibVant：移动端UI库vant UI的多语言文件，移动端项目必须要通过i18n引入；
    + 其他文件都不是项目必须，不用关系。
    **修改注意：
      1、新增： 系统公用的多语言新增，只需要在现有文件里面新增即可，不需要新增文件。
      2、修改：如无特殊需求，请不要修改已有多语言文件内容。有差异的多语言请新增。如果必须要修改，请考虑对现有项目的影响，如不确定可以联系管理员**
    <br />

  - 04-kk-component-mobile/04-kk-component：全部系统公共的多语言文件，以及UI库的多语言（我们多语言种类笔ui库多，需要提出来自己做翻译）。
    + system：全部系统公共的多语言文件，新建项目需要通过i18n引入的文件;
    + uiLib：pc端UI库element plus的多语言文件，pc项目必须要通过i18n引入；
    + uiLibVant：移动端UI库vant UI的多语言文件，移动端项目必须要通过i18n引入；
    + 其他文件都不是项目必须，不用关系。
    **修改注意：
      1、新增： 组件新增按照【首字母大写驼峰】的驼峰命名方式命名组件名称即可。
      2、修改： 现有组件调整请联系管理员。如果必须要修改，请考虑对现有项目的兼容影**

### 3、开发规范
 + **组件名应该始终是多个单词的**，根组件 App 以及 <transition>、<component> 之类的 Vue 内置组件除外。这样做可以避免跟现有的以及未来的 HTML 元素相冲突，因为所有的 HTML 元素名称都是单个单词的。
 + 所有组件使用 SFC 或 TSX 形式编写
 + SFC 中的标签顺序统一为 
    ```
    <script>
    <template>
    <style>
    ````
    这样更符合“先声明，后使用”的逻辑
 + **凡是跟组件名相关的（组件名、文件名及在模板中使用），都必须采用大驼峰 PascalCase 形式**
 + 一个文件只允许存在一个组件
 + 与父组件紧密耦合的子组件，命名要以父组件名为前缀，例：TodoList、TodoListItem、TodoListItemButton，详见 紧密耦合的组件名称；
 + 一般化描述放前，特殊化描述放后，例：SearchButtonClear、SettingsCheckboxLaunchOnStartup
 + 起名尽量使用完整单词而不是缩写，前提是要配置好编辑器，使书写长命名的代价足够低。
 + defineProps 或 defineEmits 要么使用运行时声明，要么使用类型声明。同时使用两种声明方式会导致编译报错。
 + props 声明时使用 camelCase【**小驼峰写法，首字母小写**】，但在模板中使用时使用 kebab-case【**全部使用小写字母，单词之间使用连字符 (-) 分隔,而不是下划线或空格，例如: my-file-name**】、
 + 一定要为 v-for 设置 key，且尽量不要用 index
 + v-if 和 v-for 禁止同时出现在一个组件上
 + 始终使用指令缩写，即:、@、#，除了 v-bind="obj" 这种情况
 + 使用 hooks 代替 HOC（高阶组件）和 renderProps
 + hooks、utils、constants、types 等文件，就近（尽量深）存放，必要时才做目录提升
